# Скилл: Systematic debugging

При технической ошибке (интеграции, бэкенд, агенты):
1. Не гадать.
2. Собрать логи и факты (что, где, когда).
3. Воспроизвести проблему.
4. Определить причину (а не симптом).
5. Предложить фикс.
6. Проверить, что фикс решает проблему и не ломает другое.

Опциональный скилл — нужен профилям, которые помогают с кодом/ТЗ, не для маркетинговых задач.
