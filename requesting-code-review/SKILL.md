# Скилл: Code review

При проверке кода (бэкенд приложения, интеграции, скрипты) проверь:
- ошибки и краевые случаи;
- безопасность (секреты не в коде, валидация ввода, права доступа);
- дубли и переусложнение;
- производительность (лишние запросы, пагинация, лимиты API);
- именование и читаемость.

Верни структурированный отчёт с приоритетами. Опциональный скилл (для технической части).
